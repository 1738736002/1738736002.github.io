6.app.json配置

app.json文件用来对微信小程序进行全局配置，文件内容为一个JSON对象

属性：

1.entryPagePath

小程序默认启动页，默认为pages列表的第一项，不支持带页面路径参数。

2.pages

指定小程序组成页面，每一项对应一个页面路径。不用写后缀

3.window

用于设置小程序的状态栏、导航条、标题、窗口背景颜色

具体属性及值：

属性 类型 默认值 描述 最低版本

navigationBarTitleText string 导航栏标题文字内容
navigationStyle string default 导航栏样式，仅支持以下值： default 默认样式 custom 自定义导航栏，只保留右上角胶囊按钮。参见注 2。 微信客户端 6.6.0
backgroundColor HexColor #ffffff 窗口的背景色

tab栏，可以切换显示对应的页面

5.networkTimeout

各类网络请求的超时时间，单位毫秒

6.debug

开启debug模式，可以在控制台输出信息，可以帮助开发者更快的定位一些常见的问题。

7.functionalPages

配置启动小程序所有的插件功能页

8.subpackages

启用小程序分包加载，将小程序划分成不同的子包，在构建时打包成不同的分包，用户在使用时按需进行加载。整个小程序大小不超过16M，单个分包大小不超过2M。优化小程序首次下载启动的时间。

打包原则：

  • 按 subpackages 配置路径进行打包，subpackages 配置路径外的目录将被打包到 app（主包） 中
  • app（主包）也可以有自己的 pages（即最外层的 pages 字段）
  • tabBar 页面必须在 app（主包）内
  • 不能是其他subpackages内的子目录

11.plugins

21.lazyCodeLoading

通常情况下，在小程序启动期间，所有页面及自定义组件的代码都会进行注入，当前页面没有使用到的自定义组件和页面在注入后其实并没有被使用。

自基础库版本 2.11.1 起，小程序支持有选择地注入必要的代码，以降低小程序的启动时间和运行时内存。

当配置了这一项时，小程序仅注入当前页面需要的自定义组件和页面代码，在页面中必然不会用到的自定义组件不会被加载和初始化。

8.page页面内的js配置

1.data

data：他是页面第一次渲染使用的初始数据。

页面加载时，data 将会以JSON字符串的形式由逻辑层传至渲染层，因此data中的数据必须是可以转成JSON的类型：字符串，数字，布尔值，对象，数组。

2.生命周期函数

  1. onLoad：生命周期回调—-监听页面加载。应用：可以做异步请求数据操作。
  2. onShow：生命周期回调—-监听页面显示
  3. onReady：生命周期回调—-监听页面初次渲染完成
  4. onHide：生命周期回调—-监听页面隐藏
  5. onUnload：生命周期回调—-监听页面卸载（关闭当前页面就是卸载）

3.页面事件处理函数1

  1. onPullDownRefresh：监听用户下拉操作，获取最新的页面和数据。当处理完数据刷新后，wx.stopPullDownRefresh可以停止当前页面的下拉刷新。
  2. onReachBottom：监听用户上拉操作，上拉加载下一页数据。在触发距离内滑动期间，本事件只会被触发一次。
  3. onShareAppMessage：监听用户点击右上角转发 。
  4. onPageScroll：监听页面滚动就触发
  5. onResize：监听页面尺寸发生改变就触发，应用：小程序横竖屏切换就会触发。
  6. onTabItemTap：当前是Tab页面，点击Tab就会触发
  7. onAddToFavorites(Object object)：监听用户点击右上角菜单“收藏”按钮的行为，并自定义收藏内容。
  • 此事件处理函数需要 return 一个 Object，用于自定义收藏内容：
字段 说明 默认值
title 自定义标题 页面标题或账号名称
imageUrl 自定义图片，显示图片长宽比为 1：1 页面截图
query 自定义query字段 当前页面的query
  1. onShareTimeline()：监听右上角菜单“分享到朋友圈”按钮的行为，并自定义分享内容。暂时只支持android平台
  2. onTabItemTap(Object object):点击tab时触发

9.6条件渲染

1.wx:if

2.block wx:if

因为 wx:if 是一个控制属性，需要将它添加到一个标签上。如果要一次性判断多个组件标签，可以使用一个 <block/> 标签将多个组件包装起来，并在上边使用 wx:if 控制属性。

注意： <block/> 并不是一个组件，它仅仅是一个包装元素，不会在页面中做任何渲染，只接受控制属性

区别wx:if和hidden：一般来说，wx:if 有更高的切换消耗而 hidden 有更高的初始渲染消耗。因此，如果需要频繁切换的情景下，用 hidden 更好，如果在运行时条件不大可能改变则 wx:if 较好。

9.7列表渲染

1.wx:for

在组件上使用 wx:for 控制属性绑定一个数组，即可使用数组中各项的数据重复渲染该组件。

默认数组的当前项的下标变量名默认为 index，数组当前项的变量名默认为 item

  • 使用 wx:for-item 可以指定数组当前元素的变量名
  • 使用 wx:for-index 可以指定数组当前下标的变量名

2.block wx:for

3.wx:key

如果列表中项目的位置会动态改变或者有新的项目添加到列表中，并且希望列表中的项目保持自己的特征和状态（如 input 中的输入内容，switch 的选中状态），需要使用 wx:key 来指定列表中项目的唯一的标识符。

wx:key 的值以两种形式提供

  1. 字符串，代表在 for 循环的 array 中 item 的某个 property，该 property 的值需要是列表中唯一的字符串或数字，且不能动态改变。
  2. 保留关键字 *this 代表在 for 循环中的 item 本身，这种表示需要 item 本身是一个唯一的字符串或者数字。

注意： 1. 花括号和引号之间如果有空格，将最终被解析成为(空格)字符串并渲染。

​ 2. 当 wx:for 的值为字符串时，会将字符串解析成字符串数组

13.自定义组件

使用步骤：

  1. 在compontens目录下新建目录，右键新建component，创建组件
  2. 在page目录下要引入的页面的.json文件中配置
  1. 在对应的页面中的.wxml文件中使用。<组件名></组件名>

父组件向子组件传递数据：

父组件向子组件传值，通过使用标签属性传值，属性名课自定义，子组件通过js文件中的properties接受数据

子组件向父组件传值

子组件触发父组件事件，并把要传递的值作为事件函数的参数传给父组件

  1. 自定义事件：定义“事件名（itemhexo）”，绑定事件函数binditemhexo=”事件的处理函数”
  2. slot标签 也就是一个插槽。等到父组件调用子组件的时候，再传递标签过来。最终这些被传递的标签就会替换slot插槽的位置。

组件属性：

  1. data，存放组件的数据。
  2. properties，接收其他组件传递的值，是属性名到属性值的映射表。
  3. observers，对于组价数据字段的监听
  4. methods，组件内方法
  5. create，组件生命周期函数，在组件刚刚被创建的时候执行，不能调用setData。
  6. lifetimes，组件的的生命周期也可以在 lifetimes 字段内进行声明（这是推荐的方式，其优先级最高）。
  7. attached ，生命周期被触发。此时， this.data 已被初始化为组件的当前值。这个生命周期很有用，绝大多数初始化工作可以在这个时机进行。
  8. ready，组件生命周期，在组件布局完成后执行。
  9. moved，在组件节点移动到节点树另一个位置的时候执行
  10. detached ，在组件离开页面节点树后， detached 生命周期被触发。退出一个页面时，如果组件还在页面节点树中，则 detached 会被触发

推荐在liftimes中声明生命周期函数，优先级高。

14.生命周期

1.应用生命周期

  1. onLaunch：应用第一次启动就会触发的事件，应用：获取用户信息
  2. onShow：y应用被用户看到就会触发的事件，应用：对应用的数据或效果，重置。
  3. onHide：应用被隐藏触发的事件，应用：暂停或者清除定时器
  4. onError：应用的代码发生了报错的时候，就会触发，应用：收集用户信息，通过异步将信息发送到后台，通过分析发现代码错误处，进行修改。
  5. onPageNotFound：页面找不到就会触发，在应用第一次启动的时候，找不到第一个启动页面就触发。应用：如果第一个页面找不到了，可以通过JS跳转到另一个页面（wx.navigateTo({url:’’})）。

2.页面生命周期

  1. onLoad：生命周期回调—-监听页面加载。应用：可以做异步请求数据操作。
  2. onShow：生命周期回调—-监听页面显示
  3. onReady：生命周期回调—-监听页面初次渲染完成
  4. onHide：生命周期回调—-监听页面隐藏
  5. onUnload：生命周期回调—-监听页面卸载（关闭当前页面就是卸载）
